## 二、致命问题详细分析

### 问题 1：法律页面全部返回 404（最高优先级）

> ❌ **受影响页面：** `/privacy`（隐私政策）｜ `/terms`（服务条款）｜ `/cookies`（Cookie 政策）

**根本原因分析**

网站的 i18n 国际化中间件会根据用户地区自动在 URL 前加上语言前缀。当中文用户或 Google 的审核人员访问这些页面时：

1. 用户请求 `/privacy`
2. 中间件重定向到 `/zh/privacy`
3. `/zh/privacy` 在 Next.js 路由中不存在 → 返回 **404**

**为什么这会导致 AdSense 被拒**

- AdSense 政策强制要求网站拥有可正常访问的隐私政策
- 隐私政策必须明确声明使用 Google AdSense 及第三方 Cookie
- 审核人员点击页面底部的「隐私政策」链接后看到 404，申请会直接被驳回

**修复方案**

方案 A（推荐）：在 `middleware.ts` 中，将法律页面路径排除在 i18n 重定向之外：

```ts
// 在 matcher 配置中排除法律页面
export const config = {
  matcher: ['/((?!privacy|terms|cookies|api).*)']
}
```

方案 B：在 `/app/[locale]/` 目录下为 `privacy`、`terms`、`cookies` 创建对应的本地化页面。

方案 C：在 `/public/` 目录下直接放置静态 HTML 版隐私政策，绕过 Next.js 路由。
